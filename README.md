# Форма заявки

Простая страница с формой. После отправки данные приходят на указанный email.

## Поля формы

- Название компании
- Контактное лицо
- Номер телефона
- Желаемая площадь
- Доп. комментарии

## Локальный запуск

1. Установите зависимости:

```bash
npm install
```

2. Создайте файл `.env` на основе `.env.example`:

```bash
copy .env.example .env
```

3. Заполните `.env` (пароль от почты — в `SMTP_PASS`).

4. Запустите сервер:

```bash
npm start
```

5. Откройте в браузере: http://localhost:3000

## Деплой на Render

[Render](https://render.com) — бесплатный хостинг для Node.js-приложений.

### Шаг 1. Загрузите код на GitHub

Репозиторий: https://github.com/eTsusss/form-site

### Шаг 2. Создайте аккаунт на Render

1. Перейдите на https://render.com и зарегистрируйтесь (можно через GitHub).
2. Подтвердите email, если потребуется.

### Шаг 3. Создайте Web Service

1. На Dashboard нажмите **New +** → **Web Service**.
2. Подключите GitHub-аккаунт, если ещё не подключён.
3. Выберите репозиторий **eTsusss/form-site**.
4. Заполните настройки:

| Поле | Значение |
|------|----------|
| **Name** | `form-site` (или любое имя) |
| **Region** | Frankfurt (EU Central) — ближе к России |
| **Branch** | `main` |
| **Runtime** | `Node` |
| **Build Command** | `npm install` |
| **Start Command** | `npm start` |
| **Instance Type** | `Free` |

5. Прокрутите до раздела **Environment Variables** и добавьте переменные:

| Key | Value |
|-----|-------|
| `WEB3FORMS_ACCESS_KEY` | ключ с web3forms.com (см. ниже) |

> **Важно:** `PORT` на Render задаётся автоматически — не добавляйте его вручную.

> **Почему не SMTP?** Бесплатный Render [блокирует порты 25, 465 и 587](https://render.com/docs/free#other-limitations) — Mail.ru SMTP с бесплатного плана не работает. Используйте Web3Forms (HTTPS, без данных компании).

6. Нажмите **Create Web Service**.

### Шаг 4. Дождитесь деплоя

Render установит зависимости и запустит сервер (обычно 2–5 минут). Когда статус станет **Live**, сайт будет доступен по адресу вида:

```
https://form-site-xxxx.onrender.com
```

### Шаг 5. Проверьте форму

Откройте URL сервиса, заполните форму и убедитесь, что письмо пришло на `runatex.doc@mail.ru`.

### Особенности бесплатного плана Render

- Сервис «засыпает» после ~15 минут без посещений — первый запрос после простоя может занять 30–60 секунд.
- Для постоянной работы без пауз нужен платный план.

### Обновление сайта

После `git push` в репозиторий Render автоматически пересоберёт и задеплоит проект.

## Настройка Web3Forms (рекомендуется, без данных компании)

### Шаг 1. Создайте форму на web3forms.com

| Поле | Что указать |
|------|-------------|
| **Form Name** | любое имя, например `Заявка` |
| **Domain name** | `form-site-lnh6.onrender.com` |

> Указывайте **только домен**, без `https://` и без `/contact`.  
> Для локальной проверки можно указать `localhost`.

### Шаг 2. Скопируйте Access Key

На dashboard Web3Forms скопируйте ключ **целиком** (кнопка Copy).

### Шаг 3. Добавьте ключ на Render

Render → **Environment** → одна переменная:

```
WEB3FORMS_ACCESS_KEY = ваш-ключ-из-web3forms
```

> **Важно:** скопируйте ключ без опечаток — одна лишняя буква = ошибка 500.  
> SMTP-переменные (`SMTP_HOST`, `SMTP_PASS` и т.д.) можно удалить — они не нужны.

### Шаг 4. Проверьте

После перезапуска Render отправьте форму. В Network должен появиться запрос на `api.web3forms.com` со статусом **200**.

## Другие альтернативы

| Сервис | Нужна компания? | Бесплатно | Примечание |
|--------|-----------------|-----------|------------|
| [Web3Forms](https://web3forms.com) | Нет | 250/мес | **Рекомендуем** — уже встроено в проект |
| [Formspree](https://formspree.io) | Нет | 50/мес | Регистрация по email, свой form ID |
| [FormSubmit](https://formsubmit.co) | Нет | Да | Совсем без регистрации, но меньше контроля |
| [Resend](https://resend.com) | Частично | 100/день | Нужно подтвердить email отправителя |
| Brevo | Да | 300/день | Требует данные компании |
| Платный Render | — | от $7/мес | Тогда работает Mail.ru SMTP напрямую |

## Настройка Mail.ru (только для локального запуска)

SMTP работает локально, но **не на бесплатном Render**.

1. **Настройки → Пароль и безопасность → Пароли для внешних приложений** — создайте пароль.
2. Укажите его в `SMTP_PASS` в файле `.env`.

| Параметр | Значение |
|----------|----------|
| SMTP_HOST | smtp.mail.ru |
| SMTP_PORT | 465 |
| SMTP_USER | runatex.doc@mail.ru |
